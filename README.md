# Open Banking Customer Churn Analysis

Customer churn and retention analysis for a synthetic retail banking portfolio using open-banking-style activity data.

## Business Problem

Retail banks need to identify customers likely to leave before balances, salary credits, or product relationships are lost. This project analyses customer activity, product holdings, engagement, and complaints to recommend retention actions.

## Tools Used

- Python: EDA, feature engineering, churn model
- SQL: cohort analysis and churn KPI cuts
- Power BI/Tableau: customer insight dashboard requirements
- Business analysis: stakeholder summary and recommendation mapping

## Key Questions

- Which customers are most likely to churn?
- Which behavioural signals predict churn?
- Which segments should receive retention offers?
- What KPIs should customer insight teams monitor?
